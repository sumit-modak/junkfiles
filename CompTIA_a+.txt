MotherBoards
├── Chipsets
│   ├── Intel vs AMD
│   ├── North Bridge: Memory Controller Hub (MCH)
│   │   └── High data transfer devices: High Speed GPU, RAM 
│   ├── South Bridge: I/O Controller Hub (ICH)
│   │   └── Secondary Controllers: I/O Devices, HDD, USB, Ethernet, PATA, SATA, Audio Codec
│   └── DMI (Direct Media Interface) = Connects north bridge and south bridge
├── Buses
│   ├── Front Side Bus (Intel)
│   ├── Hyper Transport Bus (AMD)
│   ├── Memory Bus = The bus which connects to the memory (RAM)
│   └── Expansion Bus = All other buses which allows us to expand the capabilities of motherboards
│       ├── ISA: Industry Standerd Architecture
│       ├── PCI: Peripheral Component Interconnect

├── MotherBoard Firmware
│   ├── BIOS(Basic Input-Output System) 
│   │   ├── About: The BIOS is stored on the BIOS chip on the motherboard
│   │   │          BIOS configuration is stored on CMOS chip which is volatile and hence requires a CMOS battery
│   │   │          On modern motherboards, the CMOS chip is integrated with the RTC on the south bridge chipset
│   │   │          User MBR (Master Boot Record) partitioning scheme
│   │   ├── POST(Power On Self Test) = When a computer is turned on, the BIOS starts by performing a 
│   │   │          series of self-tests to check the hardware components connected to the system. This 
│   │   │          process is called the Power-on Self-Test (POST). It checks the CPU, memory, storage 
│   │   │          devices, and other essential components.
│   │   ├── Boot Loader = Once the POST is successful, the BIOS searches for a boot loader program in 
│   │   │          the boot sector of the primary storage device. This program contains instructions 
│   │   │          on how to load the operating system into memory
│   │   ├── Initializing the hardware = The BIOS checks and initializes all the hardware components 
│   │   │          attached to the system, including the processor, memory modules, hard drives, 
│   │   │          optical drives, and other peripherals.
│   │   ├── Providing system access = The BIOS allows the user to access the system setup utility 
│   │   │          through the BIOS setup menu by pressing a designated key during boot-up. This setup 
│   │   │          utility allows users to configure various system parameters, such as date and time, 
│   │   │          boot order, security settings, and power management options.
│   │   ├── Executing the boot process = After initializing the hardware and loading the boot loader, 
│   │   │          the BIOS jumps to the boot loader program, which then loads the operating system 
│   │   │          into memory. The operating system takes control of the computer from this point on.
│   │   ├── Storing system configuration = The BIOS also allows users to store system configuration 
│   │   │          settings in a non-volatile memory called CMOS. This memory can hold settings such 
│   │   │          as date and time, boot order, password protection, and other system settings even 
│   │   │          after the system is turned off.
│   │   └── Updating the BIOS = The BIOS firmware can also be updated or flashed to provide new features 
│   │              or fix known issues. BIOS updates are typically available from the computer manufacturer's 
│   │              website and can be installed using a special tool provided by the manufacturer.
│   ├── MBR(Master Boot Record) 
│   │   ├── About: Located in the first sector of bootable disk
│   │   │          Size = 512 bytes
│   │   │          Contains Primary Boot Loader Info and Partition Table Info
│   │   └── MBR validation check
│   └── UEFI(Unified Extensible Firmware Interface)
│       ├── About: Has user friendly, graphical user interface
│       │           Also comes with "Secure Boot" feature which stops digitally unsigned drivers
│       │           Gives the ability to access more than 2TB storage
│       │           Uses GPT (GUID Partition Table) Partitioning scheme
│       └──

Memory
├── Memory Controller = It is the go between memory and the CPU. It helps to store and retrive data
│       from the memory. It used to be within North Bridge, nowadays it usually built on CPU die.
├── Types 
│   ├── ROM: Read Only Memory = Non-Volatile
│   └── RAM: Random Access Memory = Volatile
│       ├── SRAM: Static RAM
│       └── DRAM: Dynamic RAM
│           ├── SIMM: Single In-Line Memory Module
│           ├── DIMM: Dual In-Line Memory Module
│           │   ├── SDRAM: Synchronous DRAM = 168-pins
│           │   └── DDR: Double Data Rate 
│           │       ├── DDR = 184-pins 
│           │       ├── DDR2 = 240-pins
│           │       ├── DDR3 = 240-pins
│           │       ├── DDR4
│           │       └── DDR5
│           ├── SODIMM: Small Outline DIMM
│           └── RDRAM: Rambus DRAM
├── Memory Latency: Column Address Strobe (CAS) Latency = Latency between memory controller which requests 
│       data and the data's availablity
├── Multichannel Architecture
